## Key insights

- Good functions should answer one business question.
- Reusable functions reduce duplication and make applications easier to extend.
- An orchestrator should coordinate tools rather than perform calculations itself.
- Git commits should represent meaningful milestones instead of "end of day" snapshots.
- The AI should explain evidence rather than calculate it. Python should perform deterministic calculations, while the LLM focuses on reasoning and communication.
- Agentic AI applications are built from small reusable tools that can be combined into larger workflows.

## Experiment

I refactored the Football Copilot application so that instead of manually specifying the opponent and venue, the application now discovers the next fixture and gathers all of the context required for a pre-match report.

This resulted in a workflow where a single function (`gather_match_context()`) coordinates multiple reusable tools.

## Result

- Finds the next fixture for a selected team.
- Builds the current league table.

## Interview takeaway

> Today I refactored my football analytics application into a modular architecture. Rather than writing one large function, I created a collection of reusable analysis tools that each answer a single business question, such as calculating the league table or comparing two teams. I then introduced an orchestration layer with `gather_match_context()`, which coordinates those tools to collect all of the information required for an AI-generated pre-match report. This architecture separates deterministic analysis from LLM reasoning and mirrors the way modern agentic AI systems are designed.
